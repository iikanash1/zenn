

【保存版】AWS認定クラウドプラクティショナー(CLF-C02) 主要サービス要点まとめ

## はじめに
AWS認定クラウドプラクティショナー（CLF-C02）の試験対策として、主要なサービスの要点を自分なりに整理しました。
試験直前の見直しや、各サービスの役割をパッと把握したい時に活用してください。

---

## 1. コンピューティングサービス
もっとも基本的な「計算リソース」を提供するサービスです。

| サービス名 | 要点・キーワード |
| :--- | :--- |
| **Amazon EC2** | 仮想サーバー。OSレベルから管理可能（IaaS）。 |
| **AWS Lambda** | サーバーレス。イベント実行型。管理不要（FaaS）。 |
| **Amazon ECS / EKS** | コンテナ実行環境（Docker）。ECSはAWS独自、EKSはKubernetes。 |
| **AWS Fargate** | コンテナ用のサーバーレス実行環境。サーバー管理が不要。 |
| **AWS Lightsail** | 低価格の定額制パッケージ（VPS）。小規模向け。 |

## 2. ストレージサービス
データの保存場所です。用途（ファイル、ブロック、オブジェクト）の違いが重要です。

| サービス名 | 要点・キーワード |
| :--- | :--- |
| **Amazon S3** | オブジェクトストレージ。耐久性99.11 9s。安価で無制限。 |
| **Amazon EBS** | EC2用のブロックストレージ。OSのインストール先（SSD/HDD）。 |
| **Amazon EFS** | 複数のEC2から共有できるファイルストレージ（NFS）。 |
| **Amazon S3 Glacier** | 長期保管用のアーカイブ。安価だが取り出しに時間がかかる。 |

## 3. データベースサービス
データの構造や用途に合わせて選択します。

| サービス名 | 要点・キーワード |
| :--- | :--- |
| **Amazon RDS** | リレーショナルデータベース（SQL）。Aurora, MySQL, PostgreSQL等。 |
| **Amazon Aurora** | AWS独自の高性能・高可用性データベース。 |
| **Amazon DynamoDB** | サーバーレスのNoSQLデータベース。低遅延でスケーラブル。 |
| **Amazon Redshift** | データウェアハウス（DWH）。大量データの分析用。 |
| **Amazon ElastiCache** | インメモリキャッシュ。読み取りの高速化。 |

## 4. ネットワークとコンテンツ配信
クラウド内のネットワーク構成や、外部との接続を管理します。

| サービス名 | 要点・キーワード |
| :--- | :--- |
| **Amazon VPC** | AWS上のプライベートな仮想ネットワーク領域。 |
| **Amazon Route 53** | 高可用なDNSサービス。ドメイン登録も可能。 |
| **Amazon CloudFront** | コンテンツ配信ネットワーク（CDN）。エッジロケーションで高速化。 |
| **AWS Direct Connect** | オンプレミスとAWSを繋ぐ専用線接続。 |
| **AWS Site-to-Site VPN** | インターネット経由の暗号化されたVPN接続。 |

## 5. セキュリティ・ガバナンス
試験でも頻出の「責任共有モデル」に関連する重要な項目です。

| サービス名 | 要点・キーワード |
| :--- | :--- |
| **AWS IAM** | 認証と認可。ユーザー、グループ、ロール、ポリシー。 |
| **AWS Shield** | DDoS保護サービス。Standardは無料。 |
| **AWS WAF** | Webアプリケーションファイアウォール。SQLiやXSSを防ぐ。 |
| **AWS KMS** | データの暗号化キーを管理。 |
| **AWS Secrets Manager** | パスワードやAPIキーなどの機密情報を管理・ローテーション。 |
| **AWS Artifact** | コンプライアンスレポート（監査報告書）の取得。 |

## 6. 管理・モニタリング・コスト
運用とコスト管理に関するサービスです。

| サービス名 | 要点・キーワード |
| :--- | :--- |
| **Amazon CloudWatch** | リソースの監視、アラート。パフォーマンスログ。 |
| **AWS CloudTrail** | 「誰が・いつ・何をしたか」の操作履歴を記録（監査）。 |
| **AWS Trusted Advisor** | コスト削減やセキュリティの最適化を自動アドバイス。 |
| **AWS Organizations** | 複数のAWSアカウントを一元管理・一括請求。 |
| **AWS Cost Explorer** | コストと使用状況を可視化・予測。 |

---

## 💡 試験対策のポイント
1. **責任共有モデル**：「AWSが責任を持つ範囲（クラウド**の**セキュリティ）」と「ユーザーが責任を持つ範囲（クラウド**内**のセキュリティ）」を区別する。
2. **クラウドの利点**：固定費から変動費へ、スケールメリット、スピード感、グローバル展開。
3. **Well-Architected フレームワーク**：6つの柱（運用、セキュリティ、信頼性、パフォーマンス、コスト、持続可能性）を理解する。
