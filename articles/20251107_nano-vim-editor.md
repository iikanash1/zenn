### はじめに
コマンドラインインターフェース（CLI）でテキストファイルを編集する際、様々なエディタが使われることをし知りました。 本記事では、初心者にも優しい「Nanoエディタ」と、学習コストは高いものの強力な「Vim」の基本的な使い方を解説します。この機会にそれぞれの特徴を理解していきましょう。

### 1. Nanoエディタ: とにかくシンプルに使いたいあなたへ

Nanoエディタは、その名の通り、非常にシンプルで直感的な操作が特徴です。CLIエディタが初めての方でも、迷うことなく使い始められるでしょう。

#### 1.1. Nanoの起動と基本操作
ターミナルで以下のコマンドを入力すると、Nanoエディタが起動します。

```bash
nano [ファイル名]
```

ファイル名を指定しない場合は、新規ファイルとして開かれます。

起動すると、画面下部に主要なコマンドが表示されます。
*   `^G`: ヘルプを表示
*   `^X`: 終了
*   `^O`: 保存
*   `^W`: 検索

「^」はCtrlキーを意味します。例えば、Ctrl+Xでエディタを終了できます。

#### 1.2. テキストの入力と編集
ファイルを開いたら、そのままキーボードでテキストを入力できます。

#### 1.3. ファイルの保存と終了
テキストを入力・編集後、`Ctrl+O`を押すとファイルの保存を促されます。ファイル名を確認してEnterを押すと保存が完了します。
保存後、または保存せずに終了したい場合は`Ctrl+X`を押します。変更が保存されていない場合は、保存するかどうかを確認されます。

#### 1.4. その他の便利な機能
*   **検索**: `Ctrl+W`で検索モードに入り、検索したい文字列を入力します。
*   **置換**: 検索モード中に`Alt+R`を押すと置換モードに切り替わります。

#### Nanoエディタのメリット・デメリット
*   **メリット**: 直感的で初心者でも使いやすい、学習コストが低い。
*   **デメリット**: 高度な編集機能やカスタマイズ性はVimに劣る。

### 2. Vim: 究極の効率を求めるあなたへ

Vimは、非常に多機能でカスタマイズ性が高く、熟練すればマウスなしで高速なテキスト編集が可能になります。しかし、その分学習コストは高めです。

#### 2.1. Vimの起動とモードの概念
Vimは「モード」という独特の概念を持っています。主なモードは以下の3つです。

1.  **ノーマルモード**: カーソル移動やコマンドの実行など、Vimの基本となるモードです。起動時はこのモードです。
2.  **挿入モード**: テキストの入力を行うモードです。
3.  **コマンドラインモード**: ファイルの保存や終了、検索・置換などのコマンドを実行するモードです。

ターミナルで以下のコマンドを入力すると、Vimが起動します。

```bash
vim [ファイル名]
```

#### 2.2. ノーマルモードでの基本操作
Vim起動時はノーマルモードです。
*   **カーソル移動**: `h` (左), `j` (下), `k` (上), `l` (右)
*   **挿入モードへの切り替え**:
    *   `i`: 現在のカーソル位置から挿入
    *   `a`: 現在のカーソル位置の次から挿入
    *   `o`: 現在の行の下に新しい行を挿入
    *   `O`: 現在の行の上に新しい行を挿入
*   **削除**:
    *   `x`: 1文字削除
    *   `dw`: 単語を削除 (delete word)
    *   `dd`: 現在の行を削除 (delete line)
*   **アンドゥ (元に戻す)**: `u`
*   **リドゥ (やり直し)**: `Ctrl+r`

#### 2.3. 挿入モードでのテキスト入力
挿入モードに切り替えたら、通常のテキストエディタのように文字を入力できます。
挿入モードからノーマルモードに戻るには`Esc`キーを押します。

#### 2.4. コマンドラインモードでの操作
ノーマルモードで`:`（コロン）を入力すると、画面下部にコマンドラインが表示され、コマンドラインモードに入ります。
*   **保存**: `:w` (write)
*   **終了**: `:q` (quit)
*   **保存して終了**: `:wq` または `:x`
*   **保存せずに終了**: `:q!` (変更を破棄して強制終了)
*   **検索**: `/検索文字列` (Enterで検索、`n`で次を検索、`N`で前を検索)
*   **置換**: `:s/検索文字列/置換文字列/g` (現在の行の全ての検索文字列を置換)
    *   `: %s/検索文字列/置換文字列/g` (ファイル全体の検索文字列を置換)

#### Vimのメリット・デメリット
*   **メリット**: 高速な編集、強力なカスタマイズ性、ほとんどのUNIX系システムに標準搭載。
*   **デメリット**: 学習コストが高い、慣れるまでに時間がかかる。

### 3. あなたに合ったエディタを選ぼう

NanoとVim、それぞれの特徴を理解できたでしょうか？
*   **手軽にCLIでファイルを編集したい、とにかくシンプルさが欲しい**: Nanoがおすすめです。
*   **プログラミングやシステム管理でCLIを多用する、長期的に効率を追求したい**: Vimの習得に挑戦する価値は十分にあります。

最初はNanoから始めて、CLIエディタに慣れてきたらVimに挑戦するというのも良い方法だと知りました。
この記事が、あなたのCLIエディタ選びの一助となれば幸いです。
